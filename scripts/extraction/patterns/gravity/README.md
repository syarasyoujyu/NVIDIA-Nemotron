# gravity_distance

## 厳密に言えること

- 問題文に `d = 0.5 * g * t^2` が明示されている。
- 隠れているのは各問題ごとの `g` だけ。
- 1 問ごとに例題は 3, 4, 5 個。
- 学習データでは `t` は 1.0 から 5.0 の範囲にある。

## 学習データから強く言えそうなこと

- 例題から逆算した `g` は、各問題の内部でほぼ一定。
- 観測上の `g` はおおむね 4.9 から 19.6 の範囲にある。
- よってこれは完全にパラメータ推定型の物理式問題。

## solver 用のパターン

### 1. one-example solve

- 例題 1 個から `g = 2d / t^2` を求めれば解ける。

### 2. interval-constrained g fit

- 各例題から、丸め誤差を考慮した `g` の許容区間を作る。
- それらの共通部分を取り、問題全体と両立する `g` の区間を作る。

### 3. target-consistent representative selection

- ターゲットの `answer` から逆算した `g` の区間も作る。
- 例題側の区間とターゲット側の区間に交差があれば、その交差区間の中点を代表値にする。

## 実装方針メモ

- 平均値ベースでも近い値は出るが、説明としては区間ベースの方が厳密。
- そのため今は、まず `g` の整合区間を作り、その中から代表値を選ぶ。
- さらに、その代表値でターゲットの `answer` に丸め一致するかを確認する。
- ルールは prompt に明示されているので、few-shot より deterministic solver 向き。
