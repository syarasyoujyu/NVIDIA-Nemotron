# equation_transformation

## 厳密に言えること

- これは「5 文字の入力列を別の短い列に変換する」問題。
- 1 問ごとに例題は 3, 4, 5 個。
- 入力長は常に 5。
- 出力長は 1 から 4。
- 使われる文字集合は学習データ上で 36 文字の固定アルファベット。
- 実データには少なくとも 2 つの表面サブタイプがある。
  - 記号のみで構成されるタイプ
  - 数字と記号が混在するタイプ

## ここで注意すべきこと

- このタイプは、他の 5 タイプのように「明示された単一公式」に落ちていない。
- つまり prompt だけから厳密な生成規則族を断言するのは難しい。
- 実務上は「少数例題から 5 文字列の書き換え規則を当てる program synthesis 問題」として扱うのが安全。

## solver 用の下位パターン

### 1. deletion / selection

- 入力の一部文字だけを残す。
- 順序維持のものと並べ替えを伴うものがある。

### 2. replacement / insertion

- 入力文字を別記号へ写像する。
- 追加文字が出る場合もある。

### 3. multiset-based rewrite

- 文字の並びより、出現集合や個数に依存して見える例もある。

### 4. arithmetic-looking subtype

- 数字と演算記号が混ざるタイプ。
- ただし通常の算術そのものとは限らず、記号列変換として扱う方が安全。

## 実装方針メモ

- このタイプだけは専用 solver をもう 1 段分けた方がよい。
- まず `symbol_only` と `mixed_digit_symbol` に分ける。
- その後、文字削除型・置換型・並べ替え型の優先順位で当てにいくのが自然。
