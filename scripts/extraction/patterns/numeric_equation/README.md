# numeric_equation

## 厳密に言えること

- 入力形式は `AB op CD`（AB・CD は 2 桁の整数、op は任意の記号 1 文字）。
- 演算子は標準的な算術の意味を持たない。各問題で独自のルールが定義される。
- 同一問題内では複数の演算子が混在することがある（演算子ごとにルールが異なる）。
- 出力は基本的に数字のみだが、一部の問題では演算子記号が出力に含まれる。
- 件数: 732 件。

## 実装上のパターン

- 演算子ごとのルールを few-shot 例題から帰納する必要がある。
- 典型的な変換候補:
  - 桁ごとの演算（和・差・積など）
  - 数値の連結（例: `AB op CD = ABCD`）
  - 全桁の和・差・積

## solver 用メモ

- few-shot 例題を丁寧に読んでルールを推定することが必須。
- 同じ演算子でも問題が異なればルールが異なる点に注意。
- 推論規則は未定義。今後実装予定。
