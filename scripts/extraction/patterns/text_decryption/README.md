# text_decryption

## 厳密に言えること

- 暗号文も平文も小文字英字と空白だけで構成される。
- 1 問ごとに例題が 3, 4, 5 個ある。
- ターゲット文の単語数も 3, 4, 5 のいずれか。
- 各例題で、暗号文側と平文側は単語数が一致する。
- 対応する単語どうしの文字数も一致する。

## 学習データから強く言えそうなこと

- 問題ごとに文字置換規則が変わる。
- ただし 1 問の内部では、同じ暗号文字は同じ平文文字に対応しているとみなせるケースが大半。
- したがって、基本形は「単語境界を保つ monoalphabetic substitution」とみるのが自然。
- 平文語彙は Wonderland 系の小さめの閉語彙で、観測上は 77 語程度に収まっている。

## solver 用の下位パターン

### 1. pure character substitution

- 各文字が 1 対 1 に置換される標準的な換字式。
- もっとも本命。

### 2. word-lexicon aided substitution

- 単純な文字対応だけで埋まらない場合でも、語彙候補が小さいので単語辞書で補える。
- 例えば 3 文字語で頻出の平文候補は `the`, `cat`, `key`, `map` などにかなり絞られる。

## 実装方針メモ

- まず例題全体から文字対応表を作る。
- 次に、未確定文字を含む単語を語彙制約で絞る。
- 単語長、出現位置、既知文字パターンを使うとかなり縮む。

## 注意

- これは Caesar とは限らない。
- 問題ごとに置換表そのものが変わる前提で扱う方が安全。
