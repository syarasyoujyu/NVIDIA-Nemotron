# symbol_equation

## 厳密に言えること

- 入力形式は `AB op CD`（AB・CD は記号 2 文字、op は任意の記号 1 文字）。
- 演算子は算術的な意味を持たない。各問題で独自のルールが定義される。
- 出力は常に入力（5 文字）より短い。
- 多くの場合、出力に入力にない文字が含まれる（記号の置換・変換）。
- 一部の問題（16 件）では出力文字がすべて入力の部分集合（文字の選択・除去）。
- 件数: 823 件。

## 実装上のパターン

- 演算子ごとのルールを few-shot 例題から帰納する必要がある。
- 典型的な変換候補:
  - 記号の置換（入力記号 → 別の記号へのマッピング）
  - 特定の文字の除去・選択

## solver 用メモ

- few-shot 例題からルールを推定することが必須。
- 同じ演算子でも問題が異なればルールが異なる点に注意。
- 推論規則は未定義。今後実装予定。
