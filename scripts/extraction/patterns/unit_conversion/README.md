# unit_conversion

## 厳密に言えること

- 入力はメートル表記の実数で、出力も実数。
- 1 問ごとに例題は 3, 4, 5 個。
- 学習データでは小数 1 桁または 2 桁が現れる。

## 学習データから強く言えそうなこと

- 各問題の内部では、`output / input` がほぼ一定。
- したがって基本形は `y = kx` の一次変換とみてよい。
- 観測上の倍率 `k` はおおむね 0.50 から 2.00 の範囲にある。
- 切片付き `y = kx + b` を使う必要は見えていない。

## solver 用のパターン

### 1. interval-constrained multiplicative fit

- 各例題 `x -> y` について、丸めを考慮した `k` の許容区間を作る。
- すべての例題の区間を交差させて、問題全体と両立する `k` の区間を作る。

### 2. target-consistent representative selection

- ターゲットの `answer` も満たす `k` の区間を逆算する。
- 例題側の区間とターゲット側の区間に交差があれば、その交差区間の中点を代表値にする。
- 交差がなければ、例題区間の中点を代表値にする。

## 実装方針メモ

- `mean(y / x)` のような単純平均は説明用の近似としては使えても、厳密な代表値としてはずれることがある。
- そのため、今は「例題と両立する区間」を先に作り、その中から代表値を選ぶ方針にしている。
- さらに、その代表値で実際にターゲットの `answer` に丸め一致するかをチェックする。
