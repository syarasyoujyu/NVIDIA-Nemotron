# 何をしているか
推論フローを明示化するためのプロンプト作成
問題の種類によって推論フローを定義している

`gen_prompt.py` が各問題のカテゴリに対応する generator 関数を呼び出し、
決定的な推論テキストを生成して `reasoning/<problem_id>.txt` に書き出す。
答えは `\boxed{...}` 形式で末尾に出力される。

# 推論フローの例
## numeral
ローマ数字変換（アラビア数字 → ローマ数字）。例示は参照せず、固定ルールのみを使う。

1. 変換対象の整数を確認する
2. 1000 (M), 900 (CM), 500 (D), ... 1 (I) の順に「現在値 ≥ 基数なら記号を追加して基数を引く」を繰り返す
3. 各ステップの途中計算（残余）を表示しながら記号列を組み立てる

## unit_conversion / gravity
どちらも「例示からスカラー係数を推定し、質問入力に掛ける」という構造を持つ。

### unit_conversion
`output = factor × input` の線形関係を仮定する。

1. 各例示について `factor = output / input` を筆算形式（反復減算の割り算）で計算する
2. 全例示の factor 値を列挙し、中央値（偶数個の場合は小さい方の中央）を選ぶ
3. 質問入力に中央値 factor を掛けた結果を筆算形式で計算し、小数第3位で切り捨てる

### gravity
`d = k × t²` のモデルを仮定する（自由落下距離）。

1. 各例示について `t²` を筆算形式で計算し、`k = d / t²` を筆算形式で計算する
2. 全例示の k 値を列挙し、中央値を選ぶ
3. 質問の `t²` を計算し、中央値 k と掛け合わせて答えを得る

## cipher
置換暗号の解読。

1. 全例示と質問の入力単語を列挙し、文字単位に分解して表示する
2. 各例示の（暗号語, 平文語）ペアを文字単位で対応付けし、暗号→平文の写像テーブルを構築する
3. 写像の現在状態（未割り当て文字も含む）を表示する
4. 質問の各暗号語を写像テーブルで部分復号し、未知文字は `?` で示す
5. 未知文字が残る場合、wonderland 単語リストとのパターンマッチング（長さ・字形・既知文字の整合性）で候補単語を絞り込み、最初の合致単語を採用して写像テーブルを更新する
6. 全単語を復号した結果を連結して答えを返す

## bit_manipulation
8ビットのビット操作ルール推定。

1. 各出力例のビット列を縦（ビット位置 × 例示）の列として表示し、ビットサムをハッシュとして付与する
2. 入力列について Identity（I）・NOT・定数（0/1）・XOR/OR/AND/AND-NOT/OR-NOT/XOR-NOT の各演算結果を出力列と比較し、一致するスロットを「マッチング表」に記録する
3. ビット0から始まる **左連続列**（stride=+1 で連続してマッチする候補列）と、最終ビットで終わる **右連続列** を各演算セクションで探索し、最長のものを選ぶ
4. 左右のカバー長を比較して長い側を優先し、重複する場合は短い側を切り詰める
5. 未確定スロットに対しては左右の連続列パターンを外挿（外挿値 `?`）し、各セクションの候補と照合して「完全一致」するカテゴリを選ぶ
6. 最終ルールベクトルを確定し、質問ビット列に適用して出力ビット列を計算する

## equation (numeric)
入出力例から数値方程式の変換ルールを推定する。

1. 例示の入力を正規表現 `(\d+)(\D)(\d+)` でパースし、演算子記号でグループ化する
2. 出力の形式（通常数値 / 演算子記号が末尾や先頭に付く「符号代替」形式）を検出し、必要なら出力を変換して統一した数値表現に直す
3. 質問の演算子記号を確認し、例示に同じ記号があれば対応グループを分析する（なければ最頻出記号でフォールバック）
4. 対象グループに対して (オペランド反転あり/なし) × (結果反転あり/なし) の 4 通り × (一般的な演算 → まれな演算) の順序で候補演算を試す。各演算を全例示で検証し、全て一致すれば採用する
5. 見つかったルールを質問に適用し、符号代替形式が必要なら後処理して答えを得る

## cryptarithm
記号式（5文字入力 A1A2 op B1B2）の変換ルール推定。

1. 各例示を `(A1, A2, op, B1, B2)` の形にパースし、演算子記号でグループ化する
2. 各グループについて出力が **順方向連結**（A1A2B1B2）か **逆方向連結**（B1B2A1A2）かを判定する
3. 質問の演算子記号が既知であれば対応する連結タイプを適用し、未知の場合は順方向連結をデフォルトとする
4. 決定した連結タイプで質問入力を組み立てて答えを返す
